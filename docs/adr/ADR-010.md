# ADR-010: Mission Management CLI

## Status
Accepted

## Context
FlowForge memerlukan antarmuka baris perintah (CLI) yang terotomatisasi untuk mengelola siklus hidup Misi di dalam Engineering Workspace. Manual editing di dalam folder fisik rentan terhadap error pemindahan berkas dan status inkonsistensi.

## Decision
1.  **Integrasi Subcommand**: Menambahkan modul subparser `mission` ke `main.py` yang memicu pemanggilan 5 aksi siklus hidup utama: `new`, `list`, `show`, `start`, dan `complete`.
2.  **Mission Lifecycle Manager**: Menyediakan kelas logic terenkapsulasi `MissionLifecycleManager` yang menangani validasi file, parsing skema YAML, modifikasi file fisik, dan verifikasi duplikasi ID.
3.  **State Synchronization**: Menjaga sinkronisasi berkas `PROJECT_STATE.yaml` di root workspace secara otomatis setiap kali ada transisi status misi (`start` dan `complete`).

## Consequences
- Siklus hidup Misi dapat dikelola sepenuhnya secara programatik lewat CLI.
- Meniadakan intervensi manual developer pada berkas fisik di folder `engineering/missions/`.
- Mempermudah integrasi runtime otomatisasi pada sprint mendatang.
