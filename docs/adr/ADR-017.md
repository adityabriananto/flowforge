# ADR-017: End-to-End Runtime Integration

## Status
Accepted

## Context
FlowForge telah mengimplementasikan seluruh domain rekayasa inti (Misi, Compiler, State, Session, Provider Registry). Kami memerlukan lapisan koordinasi terpadu (Runtime Integration) yang menyambungkan seluruh domain ini ke dalam satu alur eksekusi rekayasa otonom tanpa adanya kopling logis terhadap vendor LLM tertentu.

## Decision
Kami memutuskan untuk mengimplementasikan **AIRuntimeEngine** sebagai pengatur integrasi runtime:
1.  **AIRuntimeEngine**: Bertugas murni sebagai koordinator stateless yang menyambungkan alur: memilah misi ➔ compile package ➔ inisiasi sesi ➔ memanggil provider execute ➔ menyinkronkan hasil rekayasa ke Engineering State ➔ merekam log sesi immutable.
2.  **CLI Command Run Refinement**: Menghubungkan sub-command `flowforge run` agar mendukung input Mission Code secara native, disamping tetap melayani file pipeline `.ff.yaml` yang lama.
3.  **Core Feature Complete**: Memantapkan API ini sebagai batas domain akhir untuk FlowForge Core (v1.5).

## Consequences
-   **Kelebihan**: Pipa orkestrasi yang stateless, terintegrasi penuh, dan terbebas dari bias data histori obrolan AI.
-   **Kelebihan**: Integrasi dengan `PROJECT_STATE.yaml` / `ENGINEERING_STATE.yaml` memfasilitasi kelangsungan context rekayasa jangka panjang proyek.
-   **Kekurangan**: Tidak ada runtime paralel untuk beberapa sesi sekaligus pada fase ini, namun arsitektur dinilai cukup modular untuk dikembangkan di masa mendatang.
