# ADR-007: Agent Profile System

## Status
Accepted

## Context
FlowForge memerlukan abstraksi tingkat tinggi untuk merepresentasikan agen eksternal, baik yang berbasis model AI maupun asisten coding otonom (misalnya Cursor). Ketergantungan langsung ke vendor AI spesifik di domain inti (Core) akan membatasi fleksibilitas dan skalabilitas FlowForge.

Untuk mengatasi ini, kami memerlukan format deklaratif yang mendeskripsikan kapabilitas (*capabilities*), batas kapasitas (*limitations*), mode eksekusi (*execution mode*), dan metadata terkait dari agen mana pun.

## Decision
1.  **Agent Profile Domain Model**: Mengimplementasikan model `@dataclass` `AgentProfile` yang independen dari implementasi API vendor AI.
2.  **Siklus Hidup & Loader**: Menyediakan `AgentProfileLoader` untuk memuat berkas konfigurasi YAML di folder `agent_profiles/`. Loader ini memvalidasi kebenaran parameter skema (versi skema = 1, duplicate ID, numeric capability score 0-100, mode eksekusi valid).
3.  **Mode Eksekusi Valid**: Membatasi `execution_mode` ke salah satu dari: `api`, `cli`, `agentic`, `interactive`, `batch`.
4.  **Decoupled Repository & Service**: Menyediakan `AgentProfileRepository` dan `AgentProfileService` untuk manajemen lifecyle (create, read, delete, list, filter by capability).

## Consequences
- Core FlowForge 100% bebas dari referensi vendor LLM.
- Penambahan asisten/agen baru hanya memerlukan penulisan berkas konfigurasi YAML baru tanpa modifikasi kode repositori (*Open-Closed Principle*).
- Mission Compiler di masa depan dapat memetakan kebutuhan Misi ke Agen secara dinamis berdasarkan skor kapabilitas di profil agen.
