# ADR-014: Long-Term Memory and Context Restoration via Engineering State (Refined)

## Status
Accepted (Refined)

## Context
Evaluasi arsitektural dan audit kode menyarankan penyempurnaan domain model `EngineeringState` sebelum diintegrasikan dengan modul `EngineeringSession` dan `ProviderRuntime`. File kebenaran tunggal harus menggambarkan lifecycle rekayasa lengkap, kohesif, dan kaya riwayat kronologis secara memadai.

## Decision
Kami memutuskan untuk memperbarui domain dan adapter persistensi Engineering State:
1.  **Rename File**: Mengubah nama berkas data kebenaran tunggal dari `PROJECT_STATE.yaml` menjadi `ENGINEERING_STATE.yaml`.
2.  **Refaktor Agregat Modular**: Memecah `EngineeringState` root menjadi child-state kohesif seperti `ProjectState`, `WorkspaceState`, `MissionState`, `ProviderState`, `SessionState`, `KnowledgeState`, `DecisionState`, dan `TimelineState`.
3.  **Kategori Knowledge & Metadata**: Mengharuskan `KnowledgeEntry` dikelompokkan ke dalam kategori (seperti `repository`, `architecture`, `technical_debt`, `security`, `performance`, `backend`, `frontend`, `api`, `infrastructure`) dan menyisipkan metadata lengkap pada keputusan (`DecisionEntry`).
4.  **Entitas OpenQuestion & Event Log**: Mengganti primitive string pada timeline dan open questions dengan objek domain formal seperti `OpenQuestion` dan `EngineeringEvent` (sebagai event log aktivitas kronologis proyek).
5.  **ProviderUsage Terstruktur**: Melacak histori penggunaan AI provider dengan mencatat waktu aktif (`started_at`, `finished_at`) per misi.
6.  **Context Restoration**: Menyempurnakan service layer agar dapat membangun context rekayasa mandiri melalui fungsi `build_resume_context()`.

## Consequences
-   **Kelebihan**: Modularitas yang sangat tinggi. `EngineeringState` terhindar dari bahaya menjadi God Object.
-   **Kelebihan**: Timeline event log bertindak sebagai *audit trail* yang merekam persis kapan suatu misi dimulai, diselesaikan, atau ketika provider berganti.
-   **Kelebihan**: Integrasi dengan `build_resume_context()` menyajikan context handover ringkas dan bersih yang siap dikonsumsi langsung oleh agen AI mana pun untuk langsung bekerja tanpa history percakapan.
-   **Kekurangan**: Struktur serialisasi lebih kompleks dibanding versi awal, namun diimbangi dengan kejelasan skema YAML yang rapi.
