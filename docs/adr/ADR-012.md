# ADR-012: CLI UX Hardening (Mission Code Representation)

## Status
Accepted

## Context
Saat dogfooding FlowForge di proyek Laravel riil, developer terdistraksi oleh penggunaan internal UUID yang rumit di baris perintah CLI. Untuk kegunaan harian, pengembang manusia memerlukan Mission Code pendek (`PROJECT-000`) sebagai pengenal publik utama, bukan UUID.

## Decision
1.  **Mission Code Field**: Menambahkan field `code` pada domain model `Mission` untuk menyimpan identitas publik yang ramah dibaca manusia.
2.  **Transisi ID Loader**: Memperbarui `MissionLoader` agar secara otomatis mengenali jika properti `id` di YAML berupa string biasa, menjadikannya `code`, dan menghasilkan `uuid.uuid4()` baru untuk `id` model agar UUID internal tetap terjaga.
3.  **Resolusi Identitas di Lifecycle & Compile**: Memodifikasi parser compile dan lifecycle manager agar mampu melacak berkas berdasarkan properti `code`, nama file YAML, atau UUID secara hibrida.
4.  **Error yang Berpusat pada Pengguna**: Kegagalan pencarian misi menyajikan teks informatif yang memandu pengguna untuk menjalankan daftar misi:
    ```
    Mission <mission_code> not found.
    Run:
    flowforge mission list
    for available Missions.
    ```

## Consequences
- Meningkatkan kegunaan FlowForge secara radikal dalam proyek nyata.
- Identitas internal (UUID) terisolasi sepenuhnya di dalam database dan tidak terekspos ke terminal user.
