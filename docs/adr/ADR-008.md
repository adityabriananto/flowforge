# ADR-008: Mission Package Compiler System

## Status
Accepted

## Context
Orkestrasi otonom memerlukan pemetaan kebutuhan tugas dari tingkatan bisnis (Misi) ke tingkatan instruksi teknis (Prompt) untuk model kecerdasan buatan. Menerjemahkan Misi secara langsung ke model kustom LLM secara instan di domain inti mengikat FlowForge pada bias vendor LLM spesifik.

Kita membutuhkan representasi perantara (*intermediate engineering artifact*) yang menampung seluruh metadata rekayasa secara terstruktur dan ter-filter sebelum diubah ke instruksi model spesifik. Konsep ini kami sebut **Mission Package**.

## Decision
1.  **Mission Package Domain Model**: Mengimplementasikan dataclass `MissionPackage` vendor-agnostic yang menampung: summary, objective, deliverables, constraints, rules, context, references, criteria, dan DOD.
2.  **Context & Rule Selection**: Menyediakan `ContextSelector` dan `RuleSelector` untuk menyaring project files, git status, dan baris aturan dari berkas aturan (`AGENTS.md`) yang hanya relevan dengan kata kunci Misi. Langkah ini meminimalkan ukuran paket payload.
3.  **YAML Output**: Mission Package di-render ke format YAML menggunakan `MissionPackageRenderer`.
4.  **CLI Command**: Menambahkan subcommand `flowforge compile <mission_file> [--profile <profile_file>]` ke shell utama untuk kompilasi lokal otonom.

## Consequences
- Engine internal terlepas sepenuhnya dari logic prompt-generation vendor LLM.
- Payload input untuk agen dioptimasi secara dinamis (mengurangi token-bloat).
- Evaluator dan model baru dapat mengkonsumsi berkas `mission_package_<id>.yaml` sebagai input terstruktur secara langsung.
