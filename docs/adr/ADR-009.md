# ADR-009: Engineering Workspace

## Status
Accepted

## Context
FlowForge memerlukan pengaturan berkas masukan yang terstruktur dan terstandarisasi untuk meminimalkan Token-Bloat dan memisahkan aset pengerjaan rekayasa (source files) dari data sisa proses kompilasi / run yang dibuat otomatis (.flowforge/). 

## Decision
1.  **Pemisahan Folder**:
    -   `engineering/` untuk semua berkas rekayasa orisinal (Source of Truth).
    -   `.flowforge/` untuk hasil log runtime, cache, dan data temporer eksekusi.
2.  **Misi Terdistribusi**: File spesifikasi Misi YAML diatur berdasarkan tahapan otonom: `backlog/` -> `active/` -> `completed/`.
3.  **Template Terstandardisasi**: Menyediakan template yang di-version untuk Mission, RFC, ADR, Sprint, dan Review di dalam folder `engineering/missions/templates/`.
4.  **Kompilator Integration**: Default input path di dalam `MissionPackageCompiler` dialihkan langsung ke direktori `engineering/decisions/` dan `engineering/architecture/`.

## Consequences
- Struktur proyek FlowForge menjadi konsisten dan terbaca tinggi untuk developer baru.
- Pipeline orkestrasi otonom di masa depan dapat memindahkan status berkas Misi secara otomatis di filesystem lokal.
- Mencegah polusi dan kebocoran berkas runtime ke dalam repositori git.
