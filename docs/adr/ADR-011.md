# ADR-011: Smart Project Bootstrap

## Status
Accepted

## Context
FlowForge memerlukan onboarding yang lancar untuk developer baru agar dapat segera mengintegrasikan FlowForge ke dalam repositori kode warisan (*legacy repositories*). Kita membutuhkan perintah `flowforge init` yang secara otomatis mengenali karakteristik repositori, menyusun workspace rekayasa, dan mensetup misi awal tanpa intervensi manual.

## Decision
1.  **Project Signatures Detection**: Menyediakan adapter deteksi proyek (`Laravel`, `Django`, `React`, `Vue`, `SpringBoot`, `Node.js`) yang mengenali bahasa pemrograman, manajer paket, framework, dan build tool.
2.  **Metadata Files**: Menyusun `WORKSPACE.yaml` (informasi jenis proyek) dan `PROJECT_STATE.yaml` (pelacakan status misi aktif/selesai) di bawah direktori `engineering/`.
3.  **Idempotensi**: Seluruh operasi inisialisasi dilarang keras menimpa data buatan pengguna jika folder/berkas sudah ada.
4.  **Initial Mission**: Membuat misi inisiasi otomatis bernama `PROJECT-000 Repository Discovery` (kode awalan `PROJECT` dapat dikonfigurasi).

## Consequences
- Onboarding developer baru berkurang dari menitan menjadi instan via `flowforge init`.
- FlowForge secara cerdas mendeteksi lingkungan proyek tanpa konfigurasi manual yang melelahkan.
- Stabilitas repositori terjaga karena pengaman repositori Git aktif secara bawaan (kecuali flag `--force` ditambahkan).
