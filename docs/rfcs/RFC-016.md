# RFC-016: Provider Interface and Runtime Configuration Specification

## 1. Abstract
Proposal ini menetapkan antarmuka port `AIProvider`, daur hidup `AIRuntimeProviderRegistry`, serta skema formal berkas konfigurasi `providers.yaml`.

---

## 2. YAML Schema Config (`providers.yaml`)
Berkas konfigurasi eksternal mematuhi skema terstruktur berikut:

```yaml
providers:
  - name: "Claude"
    enabled: true
    command: "uv run python agents/coder.py"
    health_command: "curl -I https://api.anthropic.com/v1/messages"

  - name: "Ollama"
    enabled: true
    command: "ollama run qwen2.5-coder:7b"
    health_command: "curl -I http://localhost:11434"

  - name: "Gemini"
    enabled: false
    command: "uv run python agents/gemini_worker.py"
    health_command: "ping api.gemini.google.com"
```

---

## 3. Public API Interfaces

### 3.1. AIProvider Port
```python
class AIProvider(ABC):
    @abstractmethod
    def name(self) -> str:
        pass

    @abstractmethod
    def health(self) -> Dict[str, Any]:
        pass

    @abstractmethod
    def execute(self, mission_package: MissionPackage, engineering_state: EngineeringState) -> Dict[str, Any]:
        pass
```

### 3.2. Registry and Configuration Loader
`ProviderConfigLoader` membaca skema di atas dan secara dinamis mendaftarkan instansi provider ke `AIRuntimeProviderRegistry`. Suku pertama yang diaktifkan (`enabled: true`) akan diset secara otomatis sebagai default.
