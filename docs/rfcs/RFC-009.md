# RFC-009: Engineering Workspace Directory Specification

## 1. Abstract
Proposal ini menetapkan spesifikasi tata letak folder standardisasi (**Engineering Workspace**) dalam setiap proyek berbasis runtime FlowForge.

## 2. Directory Hierarchy
Setiap proyek FlowForge harus diinisialisasi dengan struktur direktori sebagai berikut:

```
├── engineering/               # Engineering Source of Truth
│   ├── missions/              # Declarative Mission specification files
│   │   ├── backlog/           # Unstarted missions
│   │   ├── active/            # Missions currently in execution
│   │   ├── completed/         # Finished missions
│   │   └── templates/         # Reusable engineering templates
│   ├── rfcs/                  # Requested for Comments proposals
│   ├── adrs/                  # Architecture Decision Records
│   ├── roadmap/               # Sprint targets and roadmap documents
│   ├── architecture/          # High-level architecture designs
│   └── decisions/             # Custom agents.md or engineering rules
│
└── .flowforge/                # Runtime Generated Artifacts only
    ├── packages/              # Cached third-party packages or plugins
    ├── cache/                 # Temporary execution cache
    ├── workspace/             # Dynamic isolated subprocess worker workspaces
    └── logs/                  # Raw execution audit logs
```

## 3. Transition System
Peralihan status Misi dikelola dengan memindahkan file YAML Misi antar folder missions:
`engineering/missions/backlog/` ➔ `engineering/missions/active/` ➔ `engineering/missions/completed/`

Mekanisme pemindahan file ini dilakukan secara terprogram menggunakan class `EngineeringWorkspace.move_mission_file(...)`.
