# RFC-017: End-to-End Runtime Orchestration Specification

## 1. Abstract
Proposal ini merinci spesifikasi integrasi runtime tingkat orkestrator FlowForge (`AIRuntimeEngine`), struktur `ExecutionResult` terstandarisasi, dan peta integrasi CLI `flowforge run <MISSION_CODE>`.

---

## 2. Orchestration Pipeline Flow
Daur hidup runtime eksekusi dari orkestrator adalah sebagai berikut:

```
[flowforge run MISSION_CODE]
          ↓
[AIRuntimeEngine.execute_mission]
          ↓
[MissionPackageCompiler.compile] ➔ Creates MissionPackage
          ↓
[EngineeringStateService.load_state] ➔ Loaded long-term state
          ↓
[AIRuntimeProviderRegistry.default] ➔ Resolves AI provider adapter
          ↓
[EngineeringSessionService.create_session] ➔ Creates session log (CREATED)
          ↓
[session_service.start_session] ➔ State transitions to ACTIVE
          ↓
[AIProvider.execute(package, state)] ➔ Vendor API Call
          ↓
[EngineeringSession.add_details] ➔ Artifacts, Decisions, Blockers, etc.
          ↓
[session_service.complete_session] ➔ Immutable status lock (COMPLETED)
          ↓
[state_service.sync] ➔ Appends history and updates ENGINEERING_STATE.yaml
          ↓
[Console Output] ➔ Prints human-friendly session summary export
```

---

## 3. ExecutionResult Schema Output
Data keluaran yang terstandarisasi hasil pemrosesan provider AI:
```json
{
  "status": "SUCCESS",
  "summary": "Implemented the database repository pattern.",
  "artifacts": [
    "engineering/architecture/discovery.md"
  ],
  "decisions": [
    {
      "title": "Use SQLite for metadata persistence",
      "rationale": "Requires Zero-Config offline operation",
      "artifact_reference": "docs/adr/ADR-001.md"
    }
  ],
  "files_changed": [
    "src/flowforge/adapters/database/database.py"
  ],
  "warnings": [],
  "blockers": [],
  "recommendations": [
    "Refactor connection pooling for FastAPI integration"
  ],
  "handover_summary": "The DB repository is complete. The next worker should focus on CLI integration."
}
```
Orkestrator menerjemahkan skema JSON keluaran di atas menjadi serangkaian transaksi penulisan log sesi (`EngineeringSession`) dan akumulasi memori proyek (`EngineeringState`).
