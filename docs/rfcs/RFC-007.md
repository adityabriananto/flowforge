# RFC-007: Agent Profile YAML Specification

## 1. Abstract
Proposal ini menjabarkan spesifikasi skema berkas **Agent Profile** berbasis YAML. Format ini berfungsi sebagai lapisan abstraksi antara FlowForge dan engine eksekusi kecerdasan buatan.

## 2. Specification Schema
Berkas profil agen didefinisikan dengan skema format berikut:

```yaml
version: "1" # Wajib
id: "claude-3-5" # Wajib
display_name: "Claude 3.5 Sonnet Agent" # Wajib
provider: "anthropic" # Wajib
execution_mode: "api" # Wajib (api, cli, agentic, interactive, batch)
capabilities: # Wajib (dictionary key-value berupa integer 0-100)
  coding: 95
  refactoring: 90
  reasoning: 95
limitations:
  - "no-direct-internet-access"
preferred_output: "text"
context_limit: 200000
supported_languages:
  - "python"
  - "javascript"
supported_frameworks:
  - "fastapi"
metadata:
  model_id: "claude-3-5-sonnet-20241022"
```

## 3. Validation Logic
Proses pemuatan profile memicu validasi ketat:
- **Tipe Properti**: Properti `capabilities` harus bertipe kamus mapping (*dictionary*) dengan skor numerik berkisar antara `0` s.d `100`.
- **Eksekusi**: Properti `execution_mode` harus bernilai salah satu dari mode eksekusi yang didukung.
- **UUID / ID Unik**: Duplikasi ID profil agen dilarang baik dalam satu sesi muat (*instance*) maupun secara global.
