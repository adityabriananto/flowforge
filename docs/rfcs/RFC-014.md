# RFC-014: Engineering State Schema and Service Specification

## 1. Abstract
Proposal ini menetapkan skema formal untuk `PROJECT_STATE.yaml` dan perilaku bisnis `EngineeringStateService` untuk memelihara memori jangka panjang proyek rekayasa FlowForge yang vendor-agnostic.

---

## 2. Canonical YAML Schema
Berkas `PROJECT_STATE.yaml` disimpan di bawah direktori `engineering/` dengan struktur sebagai berikut:

```yaml
version: "1"

project:
  id: "uuid-project-1234"
  name: "FlowForge Platform"

workspace:
  version: "1.0.0"
  engineering_phase: "coding"

mission:
  current_mission: "PROJECT-002"
  completed_missions:
    - "PROJECT-000"
    - "PROJECT-001"
  mission_history:
    - "PROJECT-000"
    - "PROJECT-001"
    - "PROJECT-002"

provider:
  current_provider: "Claude"
  provider_history:
    - "Gemini"
    - "Ollama"

session:
  current_session_id: "session-abc-123"
  latest_session_id: "session-xyz-987"

knowledge:
  - id: "KNOW-001"
    title: "Repository Discovery"
    reference_path: "engineering/architecture/discovery.md"
    extracted_at: "2026-07-10T08:00:00Z"

decisions:
  - id: "DEC-001"
    title: "Use SQLite for metadata persistence"
    rationale: "Requires Zero-Config offline operation"
    decided_at: "2026-07-10T08:05:00Z"

open_questions:
  - "Should we support multiple concurrent active sessions?"

blockers:
  - id: "BLK-001"
    description: "Redis server credentials are not configured in local cluster"
    created_at: "2026-07-10T08:10:00Z"

recommendations:
  - id: "REC-001"
    suggestion: "Initialize Distributed Worker test suite"
    proposed_at: "2026-07-10T08:15:00Z"

timeline:
  - id: "MILE-001"
    title: "Sprint 14 Release"
    target_date: "2026-07-15"
```

---

## 3. Context Restoration Behavior
`EngineeringStateService.resume_engineering_state` mengekspos representasi ringkas status rekayasa agar siap diinjeksikan sebagai memori kontekstual oleh AI worker berikutnya. Konteks ini memuat data proyek, fase pengerjaan, misi aktif, sisa hambatan (*blockers*), keputusan terbaru, dan rekomendasi langkah kerja selanjutnya.
