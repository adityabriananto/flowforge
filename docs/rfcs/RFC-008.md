# RFC-008: Mission Package YAML Specification

## 1. Abstract
Proposal ini menjabarkan spesifikasi skema luaran **Mission Package** berbasis YAML yang dihasilkan oleh kompiler alur kerja rekayasa FlowForge.

## 2. Compilation Inputs
Mission Package Compiler memproses parameter:
1.  **Mission**: Berkas YAML Misi v1 yang berisi deskripsi, deliverables, dan constraints.
2.  **Agent Profile**: Profil kemampuan model target untuk menyesuaikan limitasi.
3.  **Project Context**: File-file repositori dan status workspace terpilih.
4.  **Engineering Rules**: Kumpulan baris peraturan dari `AGENTS.md` yang tersaring.
5.  **Runtime State**: Status orkestrasi alur kerja aktif.

## 3. Output YAML Schema
Kompilasi menghasilkan berkas `mission_package_<id>.yaml` dengan format skema berikut:

```yaml
mission_summary: "Mission: Implement OAuth2 Authentication (Status: ACTIVE)"
objective: "Design and implement secure JWT authentication."
deliverables:
  - "User DB schema migration"
  - "JWT encryption library"
constraints:
  - "No plain-text tokens"
  - "Agent limitation: requires-paid-subscription"
relevant_rules:
  - "- Pastikan tidak ada credential yang bocor ke git logs."
relevant_context:
  sprint_status: "Active Sprint 12 (v1.2)"
  workflow_state: "ACTIVE"
relevant_references:
  - "[RFC 6749]: Details on OAuth 2.0 Auth Framework"
acceptance_criteria:
  - "User DB schema migration"
  - "JWT encryption library"
definition_of_done:
  - "UT coverage > 85%"
```
