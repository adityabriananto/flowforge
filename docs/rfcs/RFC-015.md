# RFC-015: Engineering Session Schema and Lifecycle Specification

## 1. Abstract
Proposal ini menetapkan antarmuka dan skema representasi log terstruktur `session_<id>.yaml` di bawah `.flowforge/logs/` serta aturan transisi status eksekusi.

---

## 2. Session Schema (`session_<id>.yaml`)
Format penyimpanan berkas log sesi rekayasa mematuhi spesifikasi berikut:

```yaml
version: "1"

metadata:
  session_id: "session-uuid-12345"
  provider: "Claude"
  mission: "PROJECT-002"
  started_at: "2026-07-10T08:00:00Z"
  finished_at: "2026-07-10T08:05:00Z"
  duration_seconds: 300.0
  status: "COMPLETED"

summary: "Implemented the database repository pattern for metadata indexing."

artifacts:
  - "engineering/architecture/discovery.md"

decisions:
  - title: "Use SQLite for metadata persistence"
    rationale: "Requires Zero-Config offline operation"
    artifact_reference: "docs/adr/ADR-001.md"
    decided_at: "2026-07-10T08:05:00Z"

files_changed:
  - "src/flowforge/adapters/database/database.py"

warnings:
  - "FastAPI client is not using connection pools"

open_questions:
  - title: "Support concurrent SQLite write access?"
    description: "SQLite doesn't natively support concurrent write operations"
    mission: "PROJECT-002"
    status: "open"
    created_at: "2026-07-10T08:02:00Z"

blockers: []

recommendations:
  - "Refactor connection pooling for FastAPI integration"

handover_summary: "The DB repository is complete. The next worker should focus on CLI integration."
```

---

## 3. Transition Rule Machine
Status sesi rekayasa dikendalikan melalui mesin status dengan aturan transisi:

```
[CREATED] ➔ [ACTIVE] ➔ [COMPLETED] | [FAILED] | [CANCELLED]
```
Sesi yang berada dalam status `COMPLETED`, `FAILED`, atau `CANCELLED` dikunci secara permanen. Panggilan mutasi data (penambahan blocker, keputusan, dsb.) pada status ini akan melemparkan kesalahan `RuntimeError`.
