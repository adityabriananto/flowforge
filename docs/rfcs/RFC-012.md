# RFC-012: Mission Code Identity Specification

## 1. Abstract
Proposal ini merinci pemetaan hibrida antara UUID internal dan kode identitas eksternal (`Mission Code`) di CLI FlowForge.

## 2. Identifier Resolution Pipeline
Setiap kali CLI memproses argumen `mission_id_or_code`, ia akan melewati pencarian:
1.  **Langkah 1**: Periksa apakah ada berkas Misi di Engineering Workspace yang nama filenya sama dengan argumen (misal `PROJECT-000.yaml`).
2.  **Langkah 2**: Periksa properti `code` di dalam metadata YAML berkas-berkas Misi.
3.  **Langkah 3**: Periksa properti `id` UUID di dalam metadata YAML berkas-berkas Misi.

Sistem pemecahan masalah ini menjamin developer tidak perlu lagi menghafal atau menyalin nilai UUID.

## 3. Human-friendly Console Responses
Jika Misi dengan kode `PROJECT-999` gagal ditemukan, konsol wajib mengembalikan format standar:
```
Mission PROJECT-999 not found.
Run:
flowforge mission list
for available Missions.
```
