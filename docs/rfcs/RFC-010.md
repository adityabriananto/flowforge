# RFC-010: Mission Management CLI Command Specification

## 1. Abstract
Proposal ini merinci spesifikasi parameter input, visualisasi konsol, dan perilaku keluaran dari sistem perintah baris utama (**Mission Management CLI**).

## 2. CLI Interface Commands
Berikut adalah deskripsi sintaksis CLI perintah manajemen misi:

### 2.1 Creation (`flowforge mission new`)
```bash
flowforge mission new "Implement database schema" --desc "Create User migration" [--id <uuid>]
```
**Perilaku**: Membuat file YAML baru di `engineering/missions/backlog/` berdasarkan template dengan status inisiasi `BACKLOG`.

### 2.2 Listing (`flowforge mission list`)
```bash
flowforge mission list
```
**Perilaku**: Menampilkan visualisasi konsol daftar misi yang terbagi berdasarkan tahapan backlog, active, dan completed.

### 2.3 Inspection (`flowforge mission show`)
```bash
flowforge mission show <mission_id>
```
**Perilaku**: Menampilkan ringkasan data properti detail berkas Misi.

### 2.4 State Transitions (`flowforge mission start` & `flowforge mission complete`)
```bash
flowforge mission start <mission_id>
flowforge mission complete <mission_id>
```
**Perilaku**:
- `start`: Memindahkan file dari folder `backlog/` ke `active/` dan mengubah status di dalam file menjadi `ACTIVE`.
- `complete`: Memindahkan file dari folder `active/` ke `completed/` dan mengubah status di dalam file menjadi `DONE`.
- Kedua perintah ini mensinkronkan data di dalam berkas status utama `PROJECT_STATE.yaml`.
