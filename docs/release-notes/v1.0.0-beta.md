# Release Notes: FlowForge v1.0.0-beta

Welcome to the first official public release of **FlowForge**! 

FlowForge is an Engineering Workflow Platform that orchestrates Engineering State. It separates Planning, Execution, and Review while preserving your project's Engineering Knowledge securely within your repository.

This beta release represents the culmination of our "Engineering Core" and "Product Core" development phases, which have been rigorously validated against real-world engineering projects including *BroDev Meeting Intelligence* and *BroDev Cashier*.

---

## Major Capabilities Introduced

### Engineering Core
The internal machinery that manages deterministic software engineering execution.
- **Engineering Workspace**: A standard, persistent directory (`engineering/`) within your repository that captures your long-term project memory, context, and architectural decisions.
- **Mission Lifecycle**: Structured task management where units of engineering work transition deterministically through Backlog → Active → Completed states.
- **Interactive Mission Authoring**: A guided CLI wizard that translates raw developer intent into well-defined, actionable engineering goals.
- **Mission Compiler**: Translates abstract goals and the repository's background context into an isolated `.package.yaml` artifact ready for AI execution.
- **Runtime Engine**: A provider-independent engine that executes Mission Packages predictably.
- **Human-Friendly Version Display**: We now translate raw Python packaging versions (`PEP 440` format like `1.0.0b0`) into a clear, developer-facing display version (`1.0.0-beta`) for the CLI, documentation, and user interfaces. The internal version remains the source of truth for packaging compatibility.

### Product Core
The developer experience layer ensuring FlowForge is accessible, maintainable, and distributable.
- **Standalone Installation**: Distributed cleanly as a Python package, executable out of the box via `pip install flowforge` or `uv tool install flowforge`.
- **Product Readiness**: A hardened CLI (`flowforge doctor`, `--version`), clear error handling, and robust backward compatibility checks.
- **Documentation**: Comprehensive Developer Onboarding, CLI references, Architectural guidelines, and Architectural Decision Records.
